# Receipts - CLAUDE.md

## What is Receipts
GitHub for non-coders. Voice note in, timestamped AI-structured claim out.
Captures the thinking, decisions and proof of work behind what non-technical
people build with AI. Individual proof of work + team collaboration +
org oversight.

Live at: https://get-receipts.lovable.app
Lovable project: ec5b807e-7f38-4789-a7af-facfaf52224a
GitHub (backend/SQL): https://github.com/Receipts-demo/receipts-demo

## Current Tech Stack
- Frontend: Lovable (React/TypeScript)
- Backend: Supabase (Postgres)
- Voice: Web Speech API (browser native)
- AI claim generation: Claude Sonnet via Supabase edge function (process-entry)
- Transcription roadmap: Good Tape API (GDPR compliant)
- Auth: Supabase Auth email/password — COMPLETE (April 15 2026)

## Supabase Schema
Table: entries
- id (uuid)
- owner_id (uuid, nullable)
- project_id (uuid, nullable)
- recorded_at (timestamptz)
- raw_transcript (text)
- claim (text)
- created_at (timestamptz)
- entry_type (text) - build/decision/idea/shipped/dropped/log
- tool_tags (text[])
- project_tag (text)
- user_id (uuid, nullable)
- transcript (text)
- user_name (text)

Table: waitlist
- id, email, created_at

Edge functions deployed:
- process-entry (calls Claude API, generates claim)
- transcribe-audio (Good Tape proxy - not currently used)

## Key Product Decisions Made
- Web Speech API chosen over Good Tape for demo (audio format issues
  with webm) - Good Tape is the production plan
- localStorage for multi-user without auth - replaced by Supabase Auth (April 15 2026)
- Lovable for UI, Claude Code for backend logic
- Five prompt types chosen based on real user need:
  built/learnt/solved/problem/next
- Session grouping of entries deferred to auth sprint
- Tool auto-detection by Claude preferred over manual tagging
- Renamed "My Projects" to "My Builds"
- Entry types map to grouping tabs
- Orb speaks back on short entries (browser speech synthesis now, ElevenLabs planned Sprint 3)
- ElevenLabs planned for Sprint 3 voice personality
